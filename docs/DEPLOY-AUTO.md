# Автообновление: правки на сервере без ручного терминала

После однократной настройки вы (или ассистент) просто запускаете один скрипт — изменения пушатся в GitHub и сразу выкатываются на сервер. Заходить по SSH и вводить команды вручную больше не нужно.

---

## Что делать каждый раз при изменениях

**В PowerShell из папки проекта выполнить:**

```powershell
Set-Location "c:\Users\CrazyStudio\OneDrive\Документы\Projects\mylent"
.\scripts\deploy.ps1
```

Скрипт сам:
1. Закоммитит и отправит изменения в GitHub (если есть изменения).
2. Подключится к серверу и выполнит: обновление кода с GitHub, пересборку фронта, перезапуск контейнеров.

Пароль от сервера попросит только если не настроен вход по ключу (см. ниже).

---

## Один раз: настроить вход на сервер без пароля (по желанию)

Чтобы при каждом запуске `deploy.ps1` не вводить пароль от VPS:

### 1. Проверить или создать SSH-ключ на компьютере

В PowerShell выполните:

```powershell
if (-not (Test-Path "$env:USERPROFILE\.ssh\id_rsa.pub")) { ssh-keygen -t rsa -N '""' -f "$env:USERPROFILE\.ssh\id_rsa" }
Get-Content "$env:USERPROFILE\.ssh\id_rsa.pub"
```

Если ключа не было — `ssh-keygen` спросит путь, просто нажмите Enter (будет создан стандартный). На экран выведется **публичный ключ** (строка вида `ssh-rsa AAAA...`). Скопируйте её целиком.

### 2. Добавить ключ на сервер

Подключитесь к серверу **один раз** (в PowerShell):

```powershell
ssh root@155.212.219.106
```

Введите пароль от VPS. Когда появится приглашение `root@...:~#`, выполните по очереди (вставьте **свой** ключ вместо `ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ`):

```bash
mkdir -p ~/.ssh
echo "ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ" >> ~/.ssh/authorized_keys
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
exit
```

Вместо `ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ` подставьте одну строку, которую вы скопировали на шаге 1 (начинается с `ssh-rsa`).

### 3. Проверка

Снова в PowerShell на своём компьютере:

```powershell
ssh root@155.212.219.106 "echo OK"
```

Если выведется `OK` и пароль не спросит — вход по ключу настроен. Дальше `.\scripts\deploy.ps1` будет обходиться без пароля.

## Итог

| Действие | Команда |
|----------|--------|
| Внести правки в код (вы или ассистент) | Как обычно в редакторе |
| Отправить правки на GitHub и обновить сайт на сервере | `.\scripts\deploy.ps1` в PowerShell из папки проекта |

Терминал на сервере вручную больше не нужен — всё делает скрипт.

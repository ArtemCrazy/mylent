# Выкладка MyLent онлайн на домен — подробно по шагам

Ниже всё по шагам: как открыть терминал, что ввести, **как подключиться к серверу по SSH** (с нуля) и привязать домен.

**Важно:** когда в инструкции написано «подключитесь к серверу» или «на VPS выполните» — не пропускайте шаги. Откройте раздел **«Как подключиться к серверу по SSH»** и сделайте все пункты по порядку (открыть PowerShell → ввести `ssh root@155.212.219.106` → ввести пароль). После этого все команды вводите уже в этом же окне — вы будете на сервере.

---

## Часть 1. Отправить код на GitHub (с вашего компьютера)

«Выполнить в папке проекта» значит: открыть командную строку, перейти в папку, где лежит проект MyLent, и там ввести команду.

### Шаг 1.1. Открыть терминал

1. Нажмите **Win + R**, введите **powershell** (или **cmd**), нажмите **Enter**.
2. Откроется окно терминала (синее — PowerShell, или чёрное — CMD).

### Шаг 1.2. Перейти в папку проекта

**В PowerShell** введите (кавычки оставить):

```
Set-Location "c:\Users\CrazyStudio\OneDrive\Документы\Projects\mylent"
```

**В CMD** введите:

```
cd /d "c:\Users\CrazyStudio\OneDrive\Документы\Projects\mylent"
```

Скопируйте нужную строку, вставьте в окно (правой кнопкой или Ctrl+V), нажмите **Enter**. После этого вы в папке проекта.

### Шаг 1.3. Отправить изменения на GitHub

В той же CMD, не выходя из папки проекта, введите по очереди три команды (после каждой нажимайте Enter):

```
git add -A
```
```
git commit -m "Updates"
```
```
git push
```

Если попросит логин/пароль от GitHub — введите. После `git push` код будет на GitHub.

## Часть 2. Подключиться к VPS и установить сайт

У вас есть сервер с IP **155.212.219.106**. Подключение к нему называется SSH.

---

### Как подключиться к серверу по SSH (полная инструкция)

Делайте эти шаги **каждый раз**, когда нужно что-то выполнить на сервере.

**Шаг А. Открыть терминал на вашем компьютере**

1. Нажмите **Win + R**.
2. Введите **powershell** и нажмите **Enter** (или **cmd**, если привыкли к чёрному окну).

**Шаг Б. Подключиться к серверу**

1. В открывшемся окне введите **ровно** эту команду и нажмите **Enter**:

```
ssh root@155.212.219.106
```

2. Если появится вопрос вида «Are you sure you want to continue connecting (yes/no)?» — введите **yes** и нажмите **Enter**.
3. Когда появится строка «password:» или «Пароль:» — введите **пароль от вашего VPS** (тот, что выдал хостинг при создании сервера). Символы при вводе **не отображаются** — это нормально, просто введите пароль и нажмите **Enter**.

**Шаг В. Проверить, что вы на сервере**

После успешного ввода пароля в начале строки появится что-то вроде:

```
root@dvzqnxfxai:~#
```

Это значит, что вы уже **на сервере**. Все команды, которые вы вводите дальше, выполняются на сервере, а не на вашем компьютере.

---

**Если команда `ssh` не сработала (пишет «не найдена» или unknown command):**

1. Скачайте PuTTY: https://www.putty.org/ — на странице найдите **putty.exe** и скачайте.
2. Запустите **putty.exe**.
3. В поле **Host Name (or IP address)** введите: **155.212.219.106**
4. Нажмите кнопку **Open**.
5. В чёрном окне введите логин: **root** и нажмите **Enter**.
6. Введите пароль от VPS (символы не видны) и нажмите **Enter**.

Снова должно появиться приглашение вида `root@...:~#` — вы на сервере.

---

### Шаг 2.1. (Первый раз) Запустить установку на сервере

Вы уже подключены к серверу (см. блок «Как подключиться к серверу по SSH» выше). В терминале **на сервере** введите одну длинную команду.

Скопируйте её **целиком** (от слова `apt-get` до конца строки) и вставьте в терминал. Нажмите Enter и подождите несколько минут.

```
apt-get update && apt-get install -y git && git clone https://github.com/ArtemCrazy/mylent.git /root/mylent && bash /root/mylent/scripts/server-setup.sh
```

Когда закончится, в конце будет написано что-то вроде «MyLent развёрнут» и адрес **http://155.212.219.106:3000**.

---

### Обновление сайта на сервере (после того как вы сделали git push с компьютера)

Когда вы обновили код на GitHub и нужно подтянуть изменения на сервер и перезапустить контейнеры — делайте **все шаги по порядку**.

1. **Открыть терминал на вашем компьютере**  
   Win + R → **powershell** → Enter.

2. **Подключиться к серверу**  
   Введите команду и нажмите Enter:
   ```
   ssh root@155.212.219.106
   ```
   Если спросят «Are you sure...» — введите **yes** и Enter.  
   Когда попросят пароль — введите пароль от VPS и Enter (символы не отображаются).

3. **Проверить, что вы на сервере**  
   В начале строки должно быть что-то вроде `root@...:~#`. Значит, вы на сервере.

4. **Подтянуть новый код с GitHub:**
   ```
   cd /root/mylent
   git pull
   ```
   После каждой строки нажимайте Enter.

5. **Перейти в папку docker и пересобрать/запустить контейнеры:**
   ```
   cd /root/mylent/docker
   docker compose build --no-cache frontend
   docker compose up -d
   ```
   Подождите 2–3 минуты. Затем проверьте:
   ```
   docker compose ps
   ```
   Должны быть в статусе **Up** три контейнера: postgres, backend, frontend.

6. **Выйти с сервера (по желанию):** введите **exit** и Enter — вернётесь в свой PowerShell.

Сайт будет доступен по адресу **http://155.212.219.106:3000**.

---

### Шаг 2.3. Создать пользователя для входа на сайт

Всё ещё в терминале на сервере. Введите по очереди:

```
cd /root/mylent/docker
```
Нажмите Enter.

```
docker compose exec backend python -m scripts.create_user
```
Нажмите Enter. Скрипт попросит ввести **логин** и **пароль** — это будут ваш логин и пароль для входа на сайт MyLent. Введите и запомните их.

### Шаг 2.4. Один раз привязать Telegram

Чтобы импортировать каналы, один раз нужно «войти» в Telegram с сервера:

```
docker compose exec -it backend python -m scripts.telegram_sync
```
Введите номер телефона (в формате +79...) и код из приложения Telegram. После успешного входа можно выйти (Ctrl+C или ввести exit, если попросит).

### Шаг 2.5. Включить подтягивание постов

Чтобы посты из каналов подтягивались раз в 10 минут:

```
docker compose exec -d backend python -m scripts.telegram_sync_loop
```

На этом установка на VPS закончена. Сайт доступен по адресу **http://155.212.219.106:3000**.

## Часть 3. Привязать свой домен и HTTPS

Чтобы сайт открывался по вашему домену (например **mylent.ru** или **lenta.ваш-сайт.ru**) и по **https**, на VPS нужно поставить Nginx и выпустить бесплатный сертификат.

### Шаг 3.1. Настроить домен у регистратора

У того, где вы купили домен (Reg.ru, Timeweb, и т.д.):

1. Зайдите в управление DNS домена.
2. Добавьте **A-запись**:
   - Имя (поддомен): **@** для корня (mylent.ru) или, например, **lenta** для lenta.ваш-домен.ru.
   - Значение (IP): **155.212.219.106**
   - TTL можно оставить по умолчанию.

Сохраните. Подождите 10–30 минут, пока обновится DNS.

### Шаг 3.2. Установить Nginx и сертификат на VPS

Снова подключитесь к VPS (как в части 2: `ssh root@155.212.219.106` или PuTTY). Затем выполните команды по очереди (подставьте вместо **ваш-домен.ru** свой домен):

Установка Nginx и Certbot:

```
apt-get update && apt-get install -y nginx certbot python3-certbot-nginx
```

Создать конфиг Nginx (одной командой, замените ваш-домен.ru на свой домен):

```
echo 'server {
  listen 80;
  server_name ваш-домен.ru;
  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }
  location /api {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }
}' > /etc/nginx/sites-available/mylent
```

Включить сайт и перезапустить Nginx:

```
ln -sf /etc/nginx/sites-available/mylent /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t && systemctl reload nginx
```

Выпустить сертификат HTTPS (снова подставьте свой домен):

```
certbot --nginx -d ваш-домен.ru
```

Certbot спросит email и согласие с условиями — введите и выберите «Yes». После этого сайт будет открываться по **https://ваш-домен.ru**.

### Шаг 3.3. Указать бэкенду и фронту ваш домен

Нужно сказать приложению, что оно работает по домену, чтобы правильно работали ссылки и CORS.

1. На сервере отредактировать файл с переменными окружения:

```
nano /root/mylent/.env
```

В файле найдите и измените строки (подставьте свой домен):

```
NEXT_PUBLIC_API_URL=https://ваш-домен.ru/api
CORS_ORIGINS=https://ваш-домен.ru
```

Сохранить: **Ctrl+O**, Enter. Выход: **Ctrl+X**.

2. Перезапустить контейнеры:

```
cd /root/mylent/docker
docker compose down
docker compose up -d
```

После этого заходите на сайт по **https://ваш-домен.ru** — он должен открываться и работать.
